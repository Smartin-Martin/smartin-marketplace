You are producing the weekly Finding. This is the single most important output of Compass. The page must be worth forwarding to a co-founder or a board member. A generic Finding is worse than none.

## Step 1. Gather material

Read the repository (`napadovnik/`, see the `napadovnik` skill reference `repository.md`). Look across recent `podnety.md` entries and the coverage maps. If `$ARGUMENTS` names an idea, focus there. Otherwise find the single most worthwhile thread across all ideas this week.

## Step 2. Find the one sharp thing

The Finding is not a summary. It is one specific thing the user did not fully see. A pattern across their own notes, an unchecked assumption, a connection they missed. Use their own words as evidence. If nothing this week is sharp enough to forward, say so plainly and stop. Do not ship filler.

> Tenhle týden nemám nic, co by stálo za přeposlání. Radši počkám. Sbírám dál.

## Step 3. Write the page

Follow the format in the `napadovnik` skill reference `finding-format.md` exactly. One page. Plain, operator-grade language, in the user's language. Sections: Co je nové, Z čeho to vychází, Co to znamená, Další krok. Footer is two lines: `Kept with Smartin Compass` then `Work smarter · smartin.work`.

Save it to `napadovnik/findings/{date}_finding.md` and show it to the user.

## Step 4. The edge

If the honest next step is something Compass cannot do alone (a full check against real company data, shared memory across people, bringing a colleague in), name it once, calmly, after the Finding, per `finding-format.md`. Not a nag. The user decides.

Never send the Finding anywhere yourself. The user decides whether to forward it.
